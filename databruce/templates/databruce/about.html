Welcome to Databruce, a new Bruce Springsteen database.
What is Databruce?

Databruce is a database site that tracks stats about Bruce Springsteen's live performing history. It was created by and is maintained by lilbud.

A Brief History

Development on Databruce first started in August 2024, and a semi-public beta test started in June 2025. While the "public" launch was planned for the summer, it took a bit longer until I felt that the site was almost "just exactly perfect" and "ready for primetime" so to speak. The "public" launch eventually happened in October 2025.

Why Create It?

Databruce was created for a number of reasons, chief among them being that I could create something that would fill a niche that none of the current Bruce sites did. I was inspired by the efforts of other communities like Jerrybase, ElGoose, Dripfield.pro, etc.

What I liked about those sites was the layout/design, how well they presented their data, and the features that each provided. Jerrybase and their advanced event search was a big influence, as it's a very useful feature that I'm not aware of many other sites having. The only Bruce-related site with a similar feature is KillingFloor, which hasn't been updated since January 2019.

What Sets It Apart?

Databruce was created to be a more "modern" Bruce database site in terms of design as well as features, which is what I think sets it apart. I took bits and pieces from other sites that I liked, and used them to create something new. Something that would, like I said earlier, fill a niche that none of the current Bruce sites did.

I also wanted to create a site that was closer to an actual database. One that allowed users to interact with and search/filter the data without requiring them to know SQL, and one that allowed for more than simply "viewing" the data like Brucebase.

Features

This section will briefly cover some of Databruce's features. Not comprehensive, but an overview of sorts.

Layout/Design

The sites I used as inspiration are well designed and present their data in a way that is easy to understand, and I tried to do the same with Databruce.

Searching/Filtering

The data on most pages are presented in Datatables and can be searched/filtered in any number of ways. Either basic search, or more "advanced" queries thanks to the Searchbuilder.

Advanced Event Search

Search for events by many combinations of possible parameters.

  • Date Range: Find events between two given dates, either specific days or just months.
  • Month/Day: Events on any given day in Bruce's performing history. Or even events on certain days of the week.
  • Location: Events by venue/city/state/country.
  • Band/Musicians: Find events where a specific musician/band appeared on stage with Bruce.

Additionally, there is also a setlist search. Which is a feature I am very proud of and also one that caused a great number of headaches. This can be used to find events based on specific songs being played, or even specific sequences of songs. Which when combined with the event search above makes it a very useful and powerful tool.

Given the complexity of the search feature, and the seemingly limitless possibilities for possible queries, this is always going to be a work in progress.

User Accounts

Users can create accounts on the site, and add shows to their profiles. Furthermore, stats can be viewed on the songs you've seen (or not seen, sorry in advance to the Europeans who've never gotten Rosalita live).

Credits/Closing

Before this page gets too long (and believe me it *was*), I'd like to close out with some thanks and credits. Both to sites I drew inspiration from, as well as those which acted as data sources.

First and foremost, Brucebase. This site was largely built upon their data and information, which they've tirelessly gathered over the past few decades. They are an invaluable resource, and if I've said it once I'll say it a thousand times that if it wasn't for Brucebase, there would be no Databruce.

SpringsteenLyrics. This site provided most of the information on the many bootleg recordings that are out there. While I pulled data from here, it was pretty heavily modified/corrected, using my collection and other sources to fill in gaps and add additional info.

Jerrybase: the use of Datatables, and the advanced search which I referenced quite heavily while developing my own. I used theirs as a basis to figure out how it should all work (in theory, anyway). The setlist search rows and some of the code behind them was taken from there with credit.

ElGoose/Dripfield.pro/Speedrun.com: The sites were the primary influence on the design of many pages. The latter two more heavily, as I really liked their card based/table layout and made use of that for the event pages.

Lastly, I want to thank the members of the r/brucespringsteen Discord server for being my unwilling Guinea Pigs beta testers. Databruce is the first time I've done anything like this, let alone something both public and on this sort of scale. Their help in testing and providing feedback was crucial to this site in it's current state. They were also instrumental in the development of Brucebot, which indirectly led to Databruce in many ways.

This site is always going to be a work in progress. Questions/Comments/Suggestions? I can usually be found on Discord (.lilbud), or you can reach out via the contact form here. I'll do my best to respond quickly.
