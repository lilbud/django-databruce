Welcome to Databruce
A Bruce Springsteen Concert Database
This page was last updated on February 26, 2026.
What is Databruce?
Databruce is a database for tracking stats about Bruce Springsteen's performing history. It was created by and is maintained by lilbud. The site was launched in June 2025 as a beta test, and July 2025 to the general public. In October 2025, I created a post on Greasy Lake to mark the true "public" launch of the site (actually letting people know about it).
Why?

Databruce was created for several reasons.

One was I felt like there was a niche to fill that none of the current Bruce-focused sites did. The desire was to create a more "modern" database site in line with newer database sites for other artists like the Grateful Dead (Jerrybase) and Goose (ElGoose and Dripfield.pro). All of those had more modern layouts, better presentation of information, and features that I felt were not present in any of the current Bruce focused sites.

The second goes hand-in-hand with the first. I wanted a site with the kind of advanced search capabilities that were present on Jerrybase. They have an advanced event/setlist search, allowing users to query the sites data in any number of ways. This feature was the initial spark for Databruce all the way back in October 2022 (predating even the first Databruce database). The only Bruce-focused site with a similar feature is Paolo Calvi's "[Killing Floor](https://www.brucespringsteen.it)", which has a basic event/location search. Unfortunately it hasn't been updated since January 2019, and is unlikely to get one anytime soon.

The third was simply wanting a user-friendly way to interact with the aforementioned Databruce database. Which was first created as an extension of Brucebot (a Discord bot I developed starting in early 2023). Originally, the bot relied on scraping Brucebase for it's info, but it became clear that that was not sustainable long-term. Additionally, the bot is only capable of very simple lookups, anything more advanced (read: closer to Jerrybase' search) required using the underlying SQL database. This is something of a barrier to entry for most, and isn't very accessible.

The Databruce site was my way to create an easy and user-friendly way to interact with the database. As well as getting closer to my initial goal of a Jerrybase-like site for Bruce.

I suppose there's also something to having a site I was in-control of. Where I had free reign to do whatever the hell I wanted with it, without having to be constrained by the limits of another site.

Unique Features
Layout/Design

A primary focus of Databruce was to create a better looking site, more "up to date" and inline with the sites I was inspired by. All of those are very well designed, and present their data in an easy to understand way, which I tried to emulate. This isn't to say that a site like Brucebase looks bad or doesn't present data well, but I think many would agree that it is somewhat dated visually. It also lacks any kind of mobile layout, which limits it's usefulness on anything that isn't a computer.

Searching/Filtering

Most pages have data presented in Datatables, which allows for sorting/filtering/searching the data, as well as a basic "advanced search" (made possible via the Datatables "Searchbuilder"). This allows users to construct queries to filter the displayed data, which is quite useful.

Advanced Search

The feature that started this whole endeavor in the first place. Largely inspired by Jerrybase, even using a similar layout.

This allows for searching of events by any number of and combination of parameters. Things like:

  • Date Range
  • Month/Day (or This Day in History)
  • Location
  • Band/Musicians present on stage

Additionally, there is the ability to search setlists by song or even sequence of songs (think Incident/Rosie). This is an incredibly powerful feature, especially when combined with the above event search. Due to the potentially limitless combination of queries, it should be considered a work in progress as I can't account for every possible query in my testing.


Some examples include:

  • Shows on a Saturday where Tougher Than The Rest was played ("Well it's saturday night, you're all dressed up in blue...")
  • Shows where Patti Smith was present and Because The Night was played
  • Street Fighting Man played in London ("Cause in sleepy London town, there's just no place for a street fighting man).

There is also a secondary search for searching setlist notes. Which includes everything from guests present on songs, which songs were acoustic/full band, etc.

Data

This extends to most of the data present on the site. I wanted a more comprehensive level of data that was present on sites like Brucebase. This was partially due to the focus on searching/filtering that I wanted, which did necessitate expanding what data was included in the database.

Things like Events now having tour legs and Event "runs", Songs having a count of public/private/snippet plays, and even being able to get a list of songs played on a given tour/run/leg/location. I even have plans to include venue address/coordinates, which is an ongoing effort.

Setlists in particular were a big focus. Setlists are broken up by set, and even include things like snippets, guests, sign requests and more. Songs are even marked based on whether Bruce was present, or if it was an instrumental. All this and more are now tracked as part of the setlists. That also means that they can be searched/queried with the advanced search above, rather than simply appearing as a text-based note on a page.

There is also an incomplete list of bootlegs and official releases. Eventually I plan to flesh out the bootleg data with info from the boots themselves. No ETA on this yet.

Credits and Closing

Before this page gets too long, I'd like to close with some thanks. This isn't a comprehensive list by any means, and I plan to expand the credits/sources page eventually.

First and foremost, Brucebase. This site was built upon their data and information, which was gathered tirelessly over the past few decades. Without them, there would be no Databruce.

SpringsteenLyrics, which provided info on the many bootlegs that are out there. The data was pretty heavily modified, using my collection as a secondary source. This is very much incomplete, and I plan to flesh this info out further down the line.

Jerrybase, ElGoose, Dripfield.pro. All three of these provided various inspiration for many parts of Databruce. From layout to features to presentation of data was all inspired by these three sites.

Jerrybase in particular deserves thanks, as their advanced search was what got me started on this whole quest in the first place (even if it took longer than I anticipated.) My advanced search is based heavily on theirs, being used to figure out on a conceptual level how it should work in the first place.

Sites like ElGoose and Dripfield.pro (in it's pre 7.0 version) were an influence on the layout and design of the site. Additionally, the event detail page takes heavy influence from the game pages on Speedrun.com.

Lastly, I want to thank the members of the r/brucespringsteen Discord server for being my unwilling ~~Guinea Pigs~~ beta testers. Databruce marks the first time I've done anything on this scale at all (let alone something that was to be publicly available). Their help in testing and providing feedback/ideas was crucial to this site being in the state that it is.

This site is always going to be a work in progress. If you have suggestions/questions, I can be reached via the contact form here or on Discord, Feel free to reach out!

Thanks for reading,
